6/4/2017 - Simple Contact Form!

Today's post is dedicated to how to build an incredibly easy and simple PHP based contact form. Let's dive right in.

The first thing you need is the form itself. There are three fields I deem important to collect; name, email, and message. Therefore the form html should look something like this:

<form method="POST" action="mail.php"> 
  <input type="text" name="name" />
  <input type="text" name="email" /> 
  <textarea name="message"></textarea> 
  <input type="submit" name="submit" value="Submit" /> 
</form>

The POST method means your data is sent via http request rather than via the URL. This is a more secure way of sending your data vs GET. The action set for mail.php means that the data is being pushed into a new php file I've created with that name. Once the data is passed in we have to assign it to php variables and then use the php mailing service to shoot of the message. After that we echo out a confirmation and we're done! Here is the simple php code:

<?php $name = $_POST['name'];
$email = $_POST['email'];
$message = $_POST['message'];
$formcontent="From: $name \n Message: $message";
$recipient = "emailaddress@here.com";
$subject = "Contact Form";
$mailheader = "From: $email \r\n";
mail($recipient, $subject, $formcontent, $mailheader) or die("Error!");
echo "Thank You!"; ?>

I've done a few extra things to make it work a little smoother / look better but this is a great and easy way to accomplish your own contact form! Soon I'll be delving into some form validation to prevent issues with bots and submissions with error. I hope you enjoyed the post and if you have any questions or comments please use the contact form!

6/1/2017 - Favicons!

To quote Wikipedia: A favicon (short for favorite icon), also known as a shortcut icon, website icon, tab icon, URL icon, or bookmark icon, is a file containing one or more small icons, associated with a particular website or web page. If that doesn't make sense, it's the little icon that shows up on your browser tabs (at least on the surface).

To the average developer or even non-developer this may seem like as simple and unimportant thing but in reality it is actually quite important and has quite a bit of depth. My favorite resource for all things favicon is a github somebody set up to gather any and all information they possibly could. You can view the page here.

For most situations a simple 16x16 or 32x32 px icon will cover you. You can place this in the root of your site and add the simple meta tag to call it and you're covered. Now let's say you want to fully optimize your favicon situation. Unfortunately favicons have not be well standardized except for aspect ratio 1:1. This means lots and lots of icons. Luckily there are online generators such as this one that will generate icons for all of the many devices + their different models. For instance, this site supports up to 15 different favicons for browsers, apple devices, android devices, windows devices, as well as a few fallbacks. This is great for people who want to bookmark your page to their desktop or launch page.

Now let's say you have a site that gets lots of views per day and you want to redesign your favicon. Just replace it right? Not that simple unfortunately. Most browsers have caching of the favicon and you can cause many of your returning users to see the cached version of the icon if you aren't taking the right steps. The easiest way to circumvent this is to alter your favicon call by adding the following: href="http://www.yoursite.com/favicon.ico?v=2". This tells your browser that even though the file name is the same, this is a new file.

5/31/2017 - HTML Shell pt. 2

Continuing on with my coverage of HTML Shell we'll be covering FavIcon, Google Web Fonts, External Stylesheets, Google Analytics, and JQuery implementations.

Include Favorites Icon? Favorites Icons (or favicons) are the little icon you see in the tabs of your browser pages. Over time this icon has changed both in support for transparency and resolution as well as being utilized for things such as tablets and phones. I'll be covering this in much more depth in a later post.

Include Google Web Fonts? These days it's tough to make your website stand out. One of the easier things you can do is implement web fonts! Google provides a free alternative and has many to choose from. Typically I don't use this feature from htmlshell and simply grab the google fonts code when I choose my fonts.

Include CSS Stylesheet? There are several ways to include styles on a website but there are also incorrect or inefficient ways to do it as well. Preferably you should always use an external / linked stylesheet. This allows for you to organize your css and makes for easy troubleshooting. If you cannot use external stylesheets you should use a style block either in the head or in the body. The last resort is inline styles but nobody likes those so avoid if you can.

Include Google Analytics? Google Analytics is a pretty simple and free analytical tool that allows you to see who has visited your site, from where, how long, etc. It's a very useful tool for sites targeting a specific demographic. Along with Google Web Fonts I generally just grab the code from Google Analytics instead of using the HTML Shell code insert.

Include jQuery? One of the more powerful and widely used javascript libraries. From a front end perspective jQuery simplifies code and makes it easier for non-javascript programmers to implement simple to complex scripts. This goes on every site I build.

Include Modernizr? Modernizr works similarly to the "if IE" tags from my last post. Essentially it's a script that analyzes what the end user is using technology wise and will class the html tag with the retrieved values. This can make your life much easier if you need to target specific operating systems, browsers, devices, device types, as well as whether or not they support certain technologies.

5/30/2017 - New Site! / HTML Shell pt. 1

First blog post! The purpose of this blog is simply to highlight my development process as well as to maybe help out some developers who may need some tips. I will be adding many features to the site over time and while I do so I'll be documenting it here.

For today's post I'm going to go over the first few features on HTMLShell.com. I use this site for most static html sites to generate the starter html. It simply cuts down on time and makes it a little easier to get started.

Include IE Conditional HTML Tag? An important trait for a front end developer is to understand the intricacies of each and every popular browser, it's advantages, and it's weaknesses. If your site looks great in Chrome and Firefox but has issues rendering in IE then this tool can be a good solution for you. Essentially it's looking to see which browser the end user is viewing the site in and then (if you're using IE) it will add the necessary classes. This makes it easy to fix an issue on specific IE browsers.

Include X-UA-Compatible Meta Tag? This particular tag can be explained two ways; Simply and Very Detailed. If you'd like to see the very detailed explanation you can check out this stack overflow article. For the simple explaination, this is another fix for Microsoft browsers essentially telling them what version of browser they should render the page as. This can fix issues in advance without having to use the IE conditional css stuff I've gone over above.

Include Viewport Meta Tag? This is a meta tag that tells devices how to view your site on multiple screen resolutions. Without this tag and setting images / tags to have large set widths can cause horizontal scrolling on smaller devices. This tag states up front that the page should scale to the screen.

Include Common Meta Tags? Ever since the dot com boom people have been trying to find the best and easist ways to manipulate the system to get views and in turn conversion to make money. Meta tags were developed to try to help search engines focus search and give relevant results to users. This set of meta tags have much less search indexing relevance than they used to but are still very important as this is the data that shows up on the search engine queries themselves. Be sure to always fill in a title and a description (author and keywords tend to have very little impact).

That's it for today, I'll be covering more tomorrow!
